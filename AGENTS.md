# AGENTS.md — Alma.Environment

## Project Purpose

F# library for resolving environment variables with a priority chain: exported environment variables take precedence over values defined in `.env` files. Published as NuGet package `Alma.Environment`.

## Tech Stack

- **Language:** F# (.NET 10)
- **Framework:** .NET SDK library
- **Package management:** Paket (`paket.dependencies` / `paket.references`)
- **Build system:** FAKE (F# Make) via `build.sh`
- **Testing:** Expecto (via `YoloDev.Expecto.TestSdk`)
- **Linting:** fsharplint (`fsharplint.json`)
- **CI/CD:** GitHub Actions
- **Key dependencies:** `FSharp.Core ~> 10.0`, `Feather.ErrorHandling ~> 2.0`

## Commands

```bash
# Install dependencies
dotnet tool restore && dotnet paket install

# Build
./build.sh build

# Run tests
./build.sh -t tests

# Lint
dotnet fsharplint lint Environment.fsproj

# Full pipeline (clean → build → lint → tests)
./build.sh -t tests

# Pack NuGet package
./build.sh -t release

# Publish to NuGet
./build.sh -t publish
```

## Architecture

This is a **pure library** — no services, no API, no Docker. It exposes:

- `Envs.loadFromFile` — loads variables from a `.env` file (returns `Result`)
- `Envs.getAll` — returns all loaded variables as `Map<string, string>`
- Variable priority: exported env vars > `.env` file values

## Build System (FAKE)

The build system uses FAKE with a standard target chain for libraries:

```
Clean → AssemblyInfo → Build → Lint → Tests → Release → Publish
```

**Build options:**
- `--no-clean` — skip cleaning `bin/`/`obj/` dirs (required on CI)
- `--no-lint` — run lint but don't fail on warnings

## CI/CD

- **tests.yaml** — runs on PRs and nightly (`0 3 * * *`), sets up .NET 10.x, runs `./build.sh -t tests`
- **pr-check.yaml** — blocks fixup commits from merging, runs ShellCheck on shell scripts
- **publish.yaml** — triggers on semver tags (e.g., `13.0.0`), publishes to NuGet.org using `NUGET_API_KEY` secret

## Release Process

1. Increment `<Version>` in `Environment.fsproj`
2. Update `CHANGELOG.md`
3. Commit, tag with the version number (e.g., `13.0.0`), and push

## Conventions

- F# idiomatic style with `result {}` computation expressions for error handling
- `Feather.ErrorHandling` for Result/AsyncResult patterns
- All source files under `src/`, tests under `tests/`
- `AssemblyInfo.fs` is auto-generated by the build — do not edit manually
- Compile order matters in `.fsproj` — `AssemblyInfo.fs` first, then `src/` files in dependency order

## Pitfalls

- **No Docker** — this is a pure library, no `docker-compose.yaml`
- **Compile order** — F# requires files in dependency order in `.fsproj`; adding a new file requires updating `<Compile Include>` items
- **Paket, not NuGet CLI** — use `dotnet paket install` to add dependencies, not `dotnet add package`
- **FAKE build** — the build system is a separate F# project under `build/`; `Targets.fs` is shared infrastructure across alma-oss repos
- **Test framework** — uses Expecto, not xUnit/NUnit; tests are console apps, not traditional test projects
